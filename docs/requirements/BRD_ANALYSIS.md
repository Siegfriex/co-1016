## 1. 비즈니스 개요

### 1.1 프로젝트 비전
**핵심 메시지**: "노력과 우연이 만나 예술가가 된다"

CuratorOdyssey 플랫폼과 연동된 피지컬 컴퓨팅 인터랙티브 아트워크를 통해, 플레이어가 몰입형 경험을 통해 예술가의 경력 궤적을 이해하고, 자신만의 스토리를 생성하여 유사한 실제 작가를 발견할 수 있도록 한다.

### 1.2 프로젝트 범위
- **피지컬 컴퓨팅 아트워크**: 아두이노/ESP32 기반 센서 시스템, 파이썬 백엔드, 게임 메커니즘
- **웹앱 통합**: 자동 모니터 제어, 결과 화면 자동 표시, CuratorOdyssey API 연동
- **인사이트**: 개인/집계/교차 분석 인사이트 제공

### 1.3 비즈니스 목표
1. **교육적 가치**: 예술가 경력 분석 개념을 게임을 통해 직관적으로 이해
2. **데이터 인사이트**: 게임 플레이 데이터를 통해 플레이어 커뮤니티 패턴 분석
3. **CuratorOdyssey 전환**: 피지컬 게임 경험 후 웹앱으로 자연스러운 전환 유도
4. **브랜드 확장**: CuratorOdyssey의 물리적 확장으로 브랜드 인지도 향상

- **백엔드 팀 (Python)**: 류지환 - FastAPI 서버, WebSocket 통신
- **프론트엔드 팀**: 류지환 - 웹앱 통합, 결과 화면
- **데이터 팀**: 류지환 - Firestore 스키마 설계
- **QA 팀**: 김현태 - 테스트/수용 기준 검증

### 3.2 RACI 매트릭스 요약

- **데이터 스키마**: 데이터(A), PO(R), 백엔드(C)

- **웹앱 통합**: PO(A), 프론트엔드(R), 백엔드/데이터(C)
- **AI 매칭**: PO(R), 백엔드(R), QA(A)

## 4. 성공 지표 (KPI)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 게임 완료율 | ≥80% | 세션 완료 수 / 시작 수 |
| 평균 플레이 시간 | 5-10분 | 세션 시간 추적 |
| AI 매칭 만족도 | ≥4.0/5.0 | 사용자 설문 (1-5점 척도) |
| CuratorOdyssey 전환율 | ≥30% | 결과 화면 → 웹앱 클릭 수 |
| 데이터 일관성 | ±0.5p | 점수 계산 검증 |
| 모니터 자동 켜기 성공률 | ≥95% | IR 센서 감지 → 모니터 켜기 성공률 |
| 점수 계산 정확도 | 100% | CuratorOdyssey 가중치 시스템 일치도 |

## 5. 기술 스택

### 5.1 하드웨어
- **아두이노/ESP32**: ESP32 (WiFi 내장) 또는 아두이노 Uno + WiFi 모듈
- **센서**: 조도 센서 3개, 초음파 센서 1개, IR 센서 1개

### 5.2 소프트웨어
- **백엔드**: Python 3.10+, FastAPI, pyserial, Firebase Admin SDK
- **웹앱**: React 18, D3.js 7, WebSocket 클라이언트
- **통신**: WebSocket (WSS/WS), REST API (CuratorOdyssey API)

## 6. 데이터 구조

### 6.1 주요 Firestore 컬렉션
1. **physical_game_sessions**: 게임 세션 데이터 (공 수집, 보물 상자 선택, 점수 계산, AI 매칭 결과)
2. **treasure_boxes**: 보물 상자 메타데이터 (나이대별 3개씩, 총 9개)
3. **treasure_box_combinations**: 보물 상자 조합식 참조 데이터 (27가지 조합)

### 6.2 CuratorOdyssey 데이터 구조 매핑
- **레이더 5축**: I, F, A, M, Sedu
- **선버스트 4축**: 제도, 학술, 담론, 네트워크
- **일관성 검증**: ±0.5p 허용 오차

## 7. 주요 제약사항

### 7.1 하드웨어 제약
- 아두이노/ESP32 센서 시스템 필수
- WiFi 통신 환경 필수
- 물리적 구조 제작 필요

### 7.2 소프트웨어 제약
- 파이썬 백엔드 (FastAPI, WebSocket)
- CuratorOdyssey API 호출 (기존 Firebase Functions 활용)
- Vertex AI 통합 (월 $30 한도)

### 7.3 성능 제약
- 게임 완료 시간: 5-10분
- 점수 계산 시간: <1초
- AI 매칭 시간: <30초
- 모니터 자동 켜기 지연: <2초

### 7.4 사용자 규모
- MVP: 동시 플레이어 1명 (단일 사용자)
- 일일 활성 플레이어: 10-20명 (예상)
