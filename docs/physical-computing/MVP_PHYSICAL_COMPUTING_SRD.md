## 1. 문서 개요 (Introduction)

### 1.1 목적 (Purpose)

본 문서는 피지컬 컴퓨팅 아트워크 MVP의 수용 기준(Acceptance Criteria)을 검증 가능한 조건, 성능 지표, 테스트 시나리오로 상세화합니다. MVP의 핵심 기능 검증을 위한 테스트 케이스를 정의합니다.

### 1.2 범위 (Scope)

**In Scope**:
- MVP 수용 기준의 검증 조건 정의
- 성능 지표 및 측정 방법
- 테스트 시나리오 및 케이스
- 하드웨어 테스트 방법

**Out of Scope**:
- 웹앱 UI/UX 테스트
- CuratorOdyssey API 테스트
- AI 매칭 시스템 테스트

### 1.3 대상 독자 (Audience)

- QA 엔지니어 (테스트 계획 및 실행)
- Physical Computing MVP 개발자 (수용 기준 이해)
- 하드웨어 엔지니어 (센서 정확도 검증)

## 2. MVP 수용 기준 상세 명세 (Detailed Acceptance Criteria)

### 2.1 AC-MVP-001: 배 모형 불도저 작동성

#### 검증 조건 (Testable Conditions)

1. **모터 전진 동작 검증**:
   - 양쪽 모터가 고정 속도로 전진하는지 확인
   - PWM 값이 512로 고정되어 있는지 확인
   - 모터 방향이 FORWARD인지 확인

2. **방향 조정 동작 검증**:
   - 조이스틱 X축 입력에 따라 모터 제어가 변경되는지 확인
   - 직진/좌회전/우회전 모드가 정확히 동작하는지 확인
   - 데드존이 적절히 작동하는지 확인

3. **반응 속도 검증**:
   - 조이스틱 입력부터 모터 제어까지의 시간 측정
   - 목표: <100ms

#### 성능 지표 (Performance Metrics)

| 지표 | 목표 값 | 측정 방법 |
|------|--------|----------|
| 모터 반응 속도 | <100ms | 조이스틱 입력부터 모터 PWM 출력까지 시간 측정 |
| 모터 속도 정확도 | PWM=512 ±10 | 오실로스코프로 PWM 값 측정 |
| 방향 전환 정확도 | 100% | 각 모드별 모터 동작 확인 |
| 데드존 범위 | ±50 | 조이스틱 X축 값 테스트 |

#### 테스트 시나리오 (Test Scenarios)

**TC-MVP-001-01: 직진 모드 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 모터 드라이버 연결 완료
- 조이스틱 연결 완료

**테스트 단계**:
1. 시스템 초기화
2. 조이스틱을 중앙 위치에 배치 (X축 = 2048)
3. 모터 PWM 값 확인 (GPIO 2, 4)
4. 모터 방향 확인 (GPIO 3, 5)

**예상 결과**:
- 모터 1 PWM = 512, 방향 = HIGH (FORWARD)
- 모터 2 PWM = 512, 방향 = HIGH (FORWARD)
- 배 모형이 직진

**통과 기준**: 양쪽 모터가 동일 속도로 전진

---

**TC-MVP-001-02: 좌회전 모드 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 모터 드라이버 연결 완료
- 조이스틱 연결 완료

**테스트 단계**:
1. 시스템 초기화
2. 조이스틱을 왼쪽으로 기울임 (X축 < 1998, 즉 x_offset < -50)
3. 모터 PWM 값 확인 (GPIO 2, 4)
4. 모터 방향 확인 (GPIO 3, 5)

**예상 결과**:
- 모터 1 PWM = 256 (또는 0), 방향 = LOW (REVERSE)
- 모터 2 PWM = 512, 방향 = HIGH (FORWARD)
- 배 모형이 왼쪽으로 회전

**통과 기준**: 배 모형이 왼쪽으로 회전

---

**TC-MVP-001-03: 우회전 모드 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 모터 드라이버 연결 완료
- 조이스틱 연결 완료

**테스트 단계**:
1. 시스템 초기화
2. 조이스틱을 오른쪽으로 기울임 (X축 > 2098, 즉 x_offset > 50)
3. 모터 PWM 값 확인 (GPIO 2, 4)
4. 모터 방향 확인 (GPIO 3, 5)

**예상 결과**:
- 모터 1 PWM = 512, 방향 = HIGH (FORWARD)
- 모터 2 PWM = 256 (또는 0), 방향 = LOW (REVERSE)
- 배 모형이 오른쪽으로 회전

**통과 기준**: 배 모형이 오른쪽으로 회전

---

**TC-MVP-001-04: 반응 속도 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 모터 드라이버 연결 완료
- 조이스틱 연결 완료
- 오실로스코프 또는 로직 애널라이저 준비

**테스트 단계**:
1. 시스템 초기화
2. 조이스틱을 중앙에서 왼쪽으로 급격히 이동
3. GPIO 1 (조이스틱 X축) 변화 시점 기록 (t1)
4. GPIO 2 (모터 우측 PWM) 또는 GPIO 8 (모터 좌측 PWM) 변화 시점 기록 (t2)
5. 반응 시간 계산: t2 - t1

**예상 결과**:
- 반응 시간 <100ms

**통과 기준**: 반응 시간이 100ms 이하

### 2.2 AC-MVP-002: 공 포집 검증

#### 검증 조건 (Testable Conditions)

1. **공 포집 성공률 측정**:
   - 각 공 종류별로 10회 포집 시도
   - 성공 횟수 / 전체 시도 횟수 × 100%
   - 목표: ≥80%

2. **공 빠져나가기 상황 연출**:
   - 그물망 구멍(r1, r2)으로 공이 빠져나가는지 확인
   - 탁구공은 r1 구멍으로 빠져나감
   - 골프공은 r2 구멍으로 빠져나감
   - 당구공은 그물망에 걸림

#### 성능 지표 (Performance Metrics)

| 지표 | 목표 값 | 측정 방법 |
|------|--------|----------|
| 탁구공 포집 성공률 | ≥80% | 10회 포집 시도 중 성공 횟수 |
| 골프공 포집 성공률 | ≥80% | 10회 포집 시도 중 성공 횟수 |
| 당구공 포집 성공률 | ≥80% | 10회 포집 시도 중 성공 횟수 |
| 공 빠져나가기 정확도 | 100% | 각 공이 올바른 구멍으로 빠져나가는지 확인 |

#### 테스트 시나리오 (Test Scenarios)

**TC-MVP-002-01: 탁구공 포집 테스트**

**전제 조건**:
- 배 모형 불도저 제작 완료
- 그물망 부착 완료
- 탁구공 또는 폼보드 판떼기(40mm 지름) 준비

**테스트 단계**:
1. 탁구공을 게임 보드에 배치
2. 조이스틱으로 배 모형을 탁구공 위치로 이동
3. 그물망이 탁구공을 포집하는지 확인
4. 포집 성공 여부 기록
5. 10회 반복

**예상 결과**:
- 포집 성공률 ≥80% (10회 중 8회 이상 성공)

**통과 기준**: 포집 성공률이 80% 이상

**전제 조건**:

- 그물망 부착 완료

**TC-MVP-002-03: 당구공 포집 테스트**

**전제 조건**:
- 배 모형 불도저 제작 완료
- 그물망 부착 완료

### 2.3 AC-MVP-003: 센서 데이터 수집 정확도

#### 검증 조건 (Testable Conditions)

1. **조도 센서 읽기 정확도**:
   - 알려진 조도 값으로 테스트
   - ADC 값이 예상 범위 내인지 확인
   - 오차율 <5%

2. **초음파 센서 읽기 정확도**:
   - 알려진 거리로 테스트
   - 측정 거리가 예상 범위 내인지 확인
   - 오차율 <5%

3. **IR 센서 읽기 정확도**:
   - 배 감지 시나리오 테스트
   - 감지/미감지 정확도 확인

4. **센서 읽기 주기**:
   - 센서 읽기 간격이 100ms인지 확인

#### 성능 지표 (Performance Metrics)

| 지표 | 목표 값 | 측정 방법 |
|------|--------|----------|
| 조도 센서 읽기 주기 | 100ms ±10ms | 타임스탬프 차이 측정 |
| 조도 센서 오차율 | <5% | 알려진 값과 비교 |
| 초음파 센서 읽기 주기 | 100ms ±10ms | 타임스탬프 차이 측정 |
| 초음파 센서 오차율 | <5% | 알려진 거리와 비교 |
| IR 센서 감지 정확도 | ≥95% | 100회 테스트 중 정확히 감지된 횟수 |

#### 테스트 시나리오 (Test Scenarios)

**TC-MVP-003-01: 조도 센서 읽기 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 조도 센서 연결 완료 (GPIO 6)
- 일정한 조도 환경 준비

**테스트 단계**:
1. 시스템 초기화
2. 조도 센서 값 읽기 시작 (GPIO 6 ADC)
3. 100회 읽기 수행
4. 각 읽기 간격 측정 (타임스탬프 차이)
5. 읽기 값 범위 확인 (0-4095)

**예상 결과**:
- 읽기 주기: 100ms ±10ms
- 읽기 값 범위: 0-4095
- 값이 일정 범위 내 유지 (조도 환경이 일정한 경우)

**통과 기준**: 읽기 주기가 100ms ±10ms, 값이 유효 범위 내

---

**TC-MVP-003-02: 초음파 센서 읽기 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 초음파 센서 연결 완료 (GPIO 7, 8)
- 알려진 거리에 장애물 배치

**테스트 단계**:
1. 시스템 초기화
2. 장애물을 10cm 거리에 배치
3. 초음파 센서로 거리 측정 (GPIO 7 Trigger, GPIO 8 Echo)
4. 측정 거리 기록
5. 장애물을 20cm, 30cm로 이동하며 반복
6. 각 거리별로 10회 측정

**예상 결과**:
- 측정 거리 오차: ±5% 이내
- 예: 10cm → 9.5-10.5cm 범위
- 예: 20cm → 19-21cm 범위
- 예: 30cm → 28.5-31.5cm 범위

**통과 기준**: 측정 거리 오차가 5% 이내

---

**TC-MVP-003-03: IR 센서 읽기 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- IR 센서 연결 완료 (GPIO 12)
- 배 모형 준비

**테스트 단계**:
1. 시스템 초기화
2. 배 모형이 IR 센서 앞에 없는 상태에서 읽기 (100회)
3. 배 모형이 IR 센서 앞에 있는 상태에서 읽기 (100회)
4. 감지 정확도 계산

**예상 결과**:
- 배 모형 없을 때: detected = false (100회)
- 배 모형 있을 때: detected = true (100회)
- 감지 정확도: ≥95%

**통과 기준**: 감지 정확도가 95% 이상

---

**TC-MVP-003-04: 센서 읽기 주기 테스트**

**전제 조건**:
- ESP32-C3 보드 전원 공급 완료
- 모든 센서 연결 완료
- 타임스탬프 기록 기능 준비

**테스트 단계**:
1. 시스템 초기화
2. 메인 루프에서 센서 읽기 시작
3. 각 센서 읽기 시점의 타임스탬프 기록
4. 100회 읽기 수행
5. 읽기 간격 계산 (이전 타임스탬프와의 차이)

**예상 결과**:
- 조도 센서 읽기 간격: 100ms ±10ms
