### 필수 필드

- **문서명**: 문서의 전체 이름 (프로젝트명 포함)

- **개정 이력**: 버전별 변경 이력
- **배포 범위**: 문서를 공유하는 대상
- **변경 관리 프로세스**: 변경 관리 방법

## 버전 관리 규칙

### 버전 번호 형식

- **주요 버전.부 버전** (예: 1.0, 1.1, 2.0)
- 초기 버전: 1.0
- 주요 변경: 주 버전 증가 (1.0 → 2.0)
- 부 변경: 부 버전 증가 (1.0 → 1.1)

### 버전 업데이트 시

1. 버전 번호 증가
2. 최종 수정일 업데이트
3. 개정 이력에 변경 내용 추가
4. 상태 업데이트 (Draft → Final)
